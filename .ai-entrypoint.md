# AI Entrypoint

Este archivo define como cualquier agente debe orientarse dentro del repositorio.

## Status

- `/.ai-entrypoint.md` es solo un punto de arranque.
- La unica fuente de verdad sigue siendo `docs/project/`.
- La instruccion repo-wide sigue siendo `.github/copilot-instructions.md`.

## Read order

1. Leer este archivo para resolver la capa correcta.
2. Leer `docs/project/00-README.md`.
3. Leer `docs/project/03-TRACEABILITY-MODEL.md`.
4. Leer `docs/project/04-DOCUMENT-MAP.md`.
5. Leer `docs/project/16-generation-pack/08-COPILOT-OPERATING-MODEL.md`.
6. Leer `docs/project/16-generation-pack/09-AI-RUNTIME-POLICY.md`.
7. Leer `docs/project/16-generation-pack/10-AI-TESTING-AND-OBSERVABILITY.md`.
8. Leer `.github/copilot-instructions.md` y las instrucciones por path aplicables.

## Layer map

- `docs/project/`: Knowledge Layer canonica.
- `ai/`: capa de navegacion para operating model, contexto, orquestacion y guardrails.
- `.github/`: execution layer canonica para prompts, agents, skills, scripts y workflows.
- `.github/copilot/`: manifests y mirrors de compatibilidad derivados.

## Task routing

- Si la tarea pide reglas, trazabilidad o arquitectura: leer `docs/project/` primero.
- Si la tarea pide orientacion AI-first: leer `ai/context/README.md` y `ai/operating-model/README.md`.
- Si la tarea pide prompts, agents, skills o instructions: usar `.github/` como fuente canonica y `.github/copilot/` solo para confirmar mirrors.
- Si la tarea pide enforcement o validacion: revisar `ai/guardrails/README.md`, `.github/workflows/`, `.github/scripts/` y `.githooks/`.

## Non-negotiable rules

- Trabajar solo desde `feature/*`.
- No editar, commitear ni empujar cambios sobre `main` o `develop`.
- Toda integracion hacia ramas protegidas debe pasar por Pull Request.
- Toda tarea debe mapear `US-XXX`, `UC-XXX`, `S-XXX`, `BDD-XXX` y `TDD-S-XXX` cuando aplique.
- Actualizar primero `docs/project/` si la documentacion no soporta la ejecucion.
- Ejecutar policy review antes de editar `ai/`, `.github/` o cualquier codigo.
- Validar Conventional Commits antes de `git commit`.
